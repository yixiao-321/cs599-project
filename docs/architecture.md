# 电商后台智能数据分析平台 - 架构说明

## 1. 项目概述

本项目基于 Agent 技术构建企业级智能数据分析平台，核心能力包括：
- **智能报表生成**：LLM驱动的自然语言报告生成
- **异常检测预警**：基于规则和LLM的智能异常检测与处理建议
- **数据分析洞察**：多维度销售数据统计与AI分析

## 6. 关键技术点

### 6.1 LLM集成模式

```
Agent处理请求
    │
    ▼
需要调用LLM?
    │
    ├── 是 ──► 构建Prompt提示词
    │              │
    │              ▼
    │        调用DeepSeek API
    │              │
    │              ▼
    │        解析LLM响应
    │              │
    │              ▼
    │        整合分析结果
    │              │
    │              └──────┐
    │                     │
    └── 否 ──► 直接处理 ──┘
                         │
                         ▼
                   返回结果给用户
```

### 6.2 提示词工程

| Agent | 提示词模板 | 用途 |
|-------|-----------|------|
| ReportGenerator | DAILY_REPORT_PROMPT | 生成日报分析 |
| ReportGenerator | WEEKLY_REPORT_PROMPT | 生成周报分析 |
| AnomalyDetector | ANOMALY_DETECTION_PROMPT | 生成处理建议 |

### 6.3 性能优化

| 策略 | 实现 | 效果 |
|------|------|------|
| 连接复用 | LLM客户端单例 | 减少连接开销 |
| 异常处理 | try-catch + fallback | 保证服务可用性 |
| 异步处理 | FastAPI异步端点 | 提升并发能力 |

## 7. 数据模型

### 7.1 核心实体关系

```
                            ┌─────────────────┐
                            │  AlertRecord   │
                            │   (告警记录)    │
                            └────────┬────────┘
                                     │
                    ┌────────────────┴────────────────┐
                    │                                 │
        ┌───────────▼───────────┐         ┌───────────▼───────────┐
        │    SalesRecord        │         │    InventoryRecord    │
        │    (销售记录)         │         │    (库存记录)         │
        └───────────┬───────────┘         └───────────┬───────────┘
                    │                                 │
         ┌──────────┼──────────┐          ┌──────────┼──────────┐
         ▼          ▼          ▼          ▼          ▼          ▼
       order_id product_name category   stock_id stock_name threshold
       quantity unit_price   total_amount stock_quantity last_updated
       customer_id sale_date  status
```

### 7.2 模型字段说明

**SalesRecord**（销售记录）：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| order_id | String | 订单ID（唯一） |
| product_name | String | 商品名称 |
| category | String | 商品分类 |
| quantity | Integer | 销售数量 |
| unit_price | Float | 单价 |
| total_amount | Float | 订单总价 |
| sale_date | DateTime | 销售日期 |

**AlertRecord**（告警记录）：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| alert_type | String | 告警类型 |
| message | String | 告警消息 |
| severity | String | 优先级（high/medium/low） |
| created_at | DateTime | 创建时间 |
| resolved | Boolean | 是否已处理 |

## 10. 扩展建议

### 10.1 短期优化

1. **缓存层**：引入Redis缓存热点数据
2. **异步任务**：使用Celery实现异步报表生成
3. **监控告警**：接入Prometheus + Grafana

### 10.2 长期规划

1. **多Agent协作**：引入LangGraph实现Agent协作
2. **多LLM支持**：支持GPT-4、Claude等多模型
3. **预测分析**：基于历史数据的销售预测
4. **智能决策**：LLM驱动的自动化决策建议

## 11. 安全设计

- **密码加密**：BCrypt哈希存储
- **输入验证**：Pydantic模型验证
- **异常处理**：全局异常捕获与日志记录
- **API限流**：防止恶意请求
