# 电商后台自动报表与异常预警系统 - 架构说明

## 1. 项目概述

本项目基于 Agent 技术构建企业级电商数据分析平台，实现智能报表生成、异常检测与预警功能。

## 3. 核心工作流

### 3.1 报表生成工作流

```
开始 → 销售分析 → 异常检测 → 生成可视化 → 生成报告 → 结束
     ↓           ↓           ↓              ↓
  Sales      Anomaly    Visualization    Report
 Analyzer    Detector       Agent       Generator
```

### 3.2 智能问答工作流

```
用户提问 → 缓存查询 → LLM调用 → 返回回答
            ↓                ↓
         命中缓存          未命中
            ↓                ↓
        直接返回          生成回答
                            ↓
                       存入缓存
```

## 4. 关键技术点

### 4.1 Agent架构

- **SalesAnalyzer**: 多维度销售数据统计与分析
- **AnomalyDetector**: 基于规则和LLM的异常检测
- **ReportGenerator**: LLM驱动的自然语言报告生成
- **VisualizationAgent**: 图表数据生成与可视化
- **ChatAgent**: 智能问答与对话管理（单例模式 + 缓存优化）

### 4.2 性能优化

| 优化策略 | 实现方式 | 效果 |
|----------|----------|------|
| 单例模式 | ChatAgent使用__new__实现单例 | 避免重复初始化LLM |
| 缓存机制 | 常用问题预设答案 + 动态缓存 | 响应时间从151s降至4.5s |
| 快速回退 | LLM失败时使用预设fallback | 保证服务可用性 |

### 4.3 安全设计

- **密码加密**: MD5哈希存储
- **输入验证**: Pydantic模型验证
- **异常处理**: 全局异常捕获

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| order_id | String | 订单ID |
| product_name | String | 商品名称 |
| category | String | 分类 |
| quantity | Integer | 数量 |
| unit_price | Float | 单价 |
| total_amount | Float | 总价 |
| customer_id | String | 客户ID |

| 字段 | 类型 | 说明 |
|------|------|------|

## 8. 扩展建议

1. **消息队列**: 引入Redis/RabbitMQ实现异步报表生成
2. **缓存层**: 使用Redis缓存热点数据
3. **权限控制**: 引入OAuth2/JWT实现细粒度权限
4. **数据同步**: 定时任务同步外部数据源
5. **监控告警**: 接入Prometheus + Grafana监控
